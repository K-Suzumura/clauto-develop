# /plan:make - 実装計画作成

仕様書（`docs/spec.md`）から実装タスク一覧を作成してください。

## 実行内容

1. `docs/spec.md` を読み込む
2. 機能要件をタスクに分解
3. 実装順序と依存関係を整理
4. `docs/plan.md` に出力

## タスク分解の原則

### 粒度の目安
- 1タスク = 1〜4時間程度の作業量
- 1つのPRで完結できる単位
- テスト可能な最小単位

### 分類カテゴリ
1. **Setup**: 環境構築、設定
2. **Data**: データモデル、マイグレーション
3. **API**: バックエンドAPI実装
4. **UI**: フロントエンド実装
5. **Test**: テスト作成
6. **Docs**: ドキュメント
7. **Refactor**: リファクタリング
