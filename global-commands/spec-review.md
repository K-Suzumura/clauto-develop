# /spec:review - 仕様書レビュー

`docs/spec.md` の品質チェックを行い、問題点を報告してください。

## 実行内容

1. `docs/spec.md` を読み込む
2. 以下のチェック項目に基づいてレビュー
3. 問題点を箇条書きで出力

## チェック項目

### 1. 完全性チェック
- [ ] 未記入の TODO コメントが残っていないか
- [ ] 必須セクションがすべて埋まっているか
- [ ] 機能一覧に対応する詳細仕様があるか

### 2. 明確性チェック
- [ ] 曖昧な表現がないか（「など」「適切に」「必要に応じて」等）
- [ ] 主語が明確か
- [ ] 数値で表現すべき箇所が定量化されているか

### 3. 一貫性チェック
- [ ] 用語の使い方が統一されているか
- [ ] IDの採番ルールが一貫しているか
- [ ] 参照関係に矛盾がないか

### 4. 実装可能性チェック
- [ ] 技術的に実現可能な仕様か
- [ ] 必要な情報が揃っているか
- [ ] エラーケースが考慮されているか

### 5. テスト可能性チェック
- [ ] 受け入れ基準が明確か
- [ ] 期待される結果が定義されているか
- [ ] 境界条件が明示されているか

## 出力形式

```markdown
# 仕様書レビュー結果

## サマリー
- **判定**: PASS / NEEDS REVISION / FAIL
- **Critical**: X件
- **Warning**: X件
- **Info**: X件

## Critical（必須修正）
- [ ] [問題1]: [説明] (セクション: X.X)
- [ ] [問題2]: [説明] (セクション: X.X)

## Warning（推奨修正）
- [ ] [問題1]: [説明] (セクション: X.X)

## Info（参考情報）
- [情報1]: [説明]

## 良い点
- [良い点1]
- [良い点2]

## 次のステップ
1. Critical の問題を修正
2. Warning の問題を検討
3. 修正後、再度 `/spec:review` を実行
```

## 注意事項

- 問題点は具体的なセクション番号と共に報告
- 修正案があれば提示
- 良い点も併せて報告してバランスを取る
