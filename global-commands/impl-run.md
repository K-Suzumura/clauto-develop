# /impl:run - タスク実装実行

指定されたタスクの実装を開始してください。

## 使用方法

```
/impl:run [タスクID]
/impl:run T-001
/impl:run T-001,T-002,T-003
```

## 実行内容

1. `docs/plan.md` から指定タスクの詳細を読み込む
2. 関連する仕様を `docs/spec.md` から確認
3. 実装を実行
4. タスクの状態を更新

## 実装の流れ

### 1. 事前確認
- 依存タスクが完了しているか確認
- 必要なファイル・設定が存在するか確認
- 実装方針をユーザーに確認（必要に応じて）

### 2. 実装
- 成果物に記載されたファイルを作成/編集
- コーディング規約に従う
- 適切なコメントを追加

### 3. 検証
- 構文エラーがないか確認
- 基本的な動作確認
- 既存テストが壊れていないか確認

### 4. 完了処理
- `docs/plan.md` のタスク状態を「完了」に更新
- 実装内容のサマリーを出力

## 出力形式

```markdown
# 実装完了: T-XXX

## タスク情報
- **タスクID**: T-XXX
- **タスク名**: [タスク名]
- **カテゴリ**: [カテゴリ]

## 変更内容

### 作成したファイル
- `path/to/new-file.ext`: [概要]

### 変更したファイル
- `path/to/modified-file.ext`: [変更内容]

## 実装のポイント
- [実装上の判断1]
- [実装上の判断2]

## 検証結果
- [x] 構文エラーなし
- [x] 基本動作確認済み
- [x] 既存テストへの影響なし

## 次のステップ
- 次のタスク: T-YYY ([タスク名])
- または `/qa:full` でテスト実行
```

## 注意事項

- 1度に複数タスクを指定した場合は、依存順に実行
- 実装中に仕様の曖昧さを発見したら報告
- 大きな設計判断が必要な場合はユーザーに確認
